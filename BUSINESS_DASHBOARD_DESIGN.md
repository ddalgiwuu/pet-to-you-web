# Business Dashboard Design Document

**Version:** 1.0
**Date:** 2026-02-07
**Designer:** UX Team
**Status:** Draft for Review

---

## 1. Executive Summary

The **Business Dashboard** is designed for pet service providers (groomers, trainers, pet-sitters, walkers) to manage their operations, bookings, customers, and revenue. This dashboard differentiates from the hospital-dashboard by focusing on service-based workflows, scheduling flexibility, and customer relationship management.

### Key Differentiators from Hospital Dashboard:
- **Service-Centric**: Focus on service packages, bundles, and custom offerings
- **Flexible Scheduling**: Time-based booking slots vs. appointment-based
- **Customer-First**: Emphasis on customer profiles, loyalty programs, and repeat business
- **Mobile-Friendly**: Service providers need on-the-go access
- **Multi-Location**: Support for home visits and mobile services

## 2. Business Requirements

### 2.1 Target Users (Service Providers)

#### Primary Personas:

**1. Professional Groomer - "Sarah"**
- **Age:** 32
- **Business:** Independent grooming salon (3 employees)
- **Goals:**
  - Manage 15-20 appointments/day
  - Track grooming packages (basic wash, full groom, spa)
  - Build customer loyalty with package deals
  - Manage before/after photos for portfolio
- **Pain Points:**
  - Double bookings
  - No-shows
  - Tracking inventory (shampoos, tools)
  - Managing employee schedules

**2. Pet Trainer - "Mike"**
- **Age:** 28
- **Business:** Mobile training service (solo)
- **Goals:**
  - Schedule training sessions at client homes or parks
  - Track training progress per pet
  - Offer package deals (puppy training: 8 sessions)
  - Send training reminders and homework
- **Pain Points:**
  - Travel time between sessions
  - Session notes scattered
  - Payment collection
  - Rescheduling complexity

**3. Pet Sitter - "Emma"**
- **Age:** 35
- **Business:** Pet sitting & walking service (5 sitters)
- **Goals:**
  - Manage multiple daily visits
  - Track pet care instructions (meds, feeding times)
  - Real-time updates to pet owners
  - Manage sitter assignments
- **Pain Points:**
  - Emergency coverage
  - Key management
  - Multiple pets per household
  - Communication gaps

**4. Multi-Service Provider - "David"**
- **Age:** 40
- **Business:** Pet care center (grooming + daycare + training)
- **Goals:**
  - Cross-sell services
  - Manage facility capacity
  - Staff scheduling across services
  - Comprehensive analytics
- **Pain Points:**
  - Complex scheduling
  - Resource allocation
  - Staff coordination
  - Multi-revenue stream tracking

### 2.2 Core Features Required

#### Must-Have (MVP):
1. **Service Management**: Create/edit services, pricing, duration, descriptions
2. **Booking System**: Calendar view, slot management, status tracking
3. **Customer Database**: Pet profiles, service history, contact info
4. **Revenue Tracking**: Daily/weekly/monthly revenue, payment status
5. **Reviews Management**: View and respond to customer reviews
6. **Basic Settings**: Business hours, services, profile

#### Should-Have (Phase 2):
7. **Schedule Optimization**: Travel time, route planning (mobile services)
8. **Package Management**: Service bundles, memberships, loyalty programs
9. **Inventory Tracking**: Product usage, restock alerts
10. **Team Management**: Employee schedules, commissions, performance
11. **Customer Communication**: SMS/email reminders, thank-you notes
12. **Photo Gallery**: Before/after photos, portfolio building

#### Nice-to-Have (Phase 3):
13. **Mobile App**: Native iOS/Android for on-the-go access
14. **Advanced Analytics**: Customer lifetime value, churn prediction, popular services
15. **Integration**: Accounting software, payment gateways, marketing tools
16. **AI Features**: Demand forecasting, dynamic pricing, customer insights

### 11.2 State Management

**Use Zustand for global state:**

```typescript
// stores/businessStore.ts
import { create } from 'zustand'

interface BusinessState {
  businessInfo: BusinessInfo | null
  setBusinessInfo: (info: BusinessInfo) => void
  bookings: Booking[]
  addBooking: (booking: Booking) => void
  updateBooking: (id: string, updates: Partial<Booking>) => void
}

export const useBusinessStore = create<BusinessState>((set) => ({
  businessInfo: null,
  setBusinessInfo: (info) => set({ businessInfo: info }),
  bookings: [],
  addBooking: (booking) => set((state) => ({
    bookings: [...state.bookings, booking]
  })),
  updateBooking: (id, updates) => set((state) => ({
    bookings: state.bookings.map(b =>
      b.id === id ? { ...b, ...updates } : b
    )
  })),
}))
```

### 11.3 API Integration

**Use React Query for data fetching:**

```typescript
// hooks/useBookings.ts
import { useQuery, useMutation, useQueryClient } from '@tanstack/react-query'
import { api } from '@/lib/api'

export function useBookings(date: string) {
  return useQuery({
    queryKey: ['bookings', date],
    queryFn: () => api.bookings.getByDate(date),
    staleTime: 60000, // 1 minute
  })
}

export function useCreateBooking() {
  const queryClient = useQueryClient()

  return useMutation({
    mutationFn: (booking: CreateBookingInput) =>
      api.bookings.create(booking),
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ['bookings'] })
    },
  })
}
```
